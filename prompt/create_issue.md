# 改善 issue 作成補助テンプレート

このファイルは補助テンプレートです。作業前に必ず `README.md`、`AGENTS.md`、`docs/architecture.md`、`docs/ai-execution.md` を確認してください。

## 目的

- 既存 issue を確認した上で、新規改善 issue を起票する
- issue テンプレートの必須項目を満たす
- 重複や粒度不整合を避ける

## 事前確認

1. `gh issue list --limit 200 --json number,title,body,labels,state`
2. 必要に応じて `gh issue view <issue-number>`
3. 関連コード、関連 docs
4. issue template が導入済みなら、その必須項目

## 作成ルール

- タイトルと本文は日本語で書く
- `Summary` `Background` `Scope` `Acceptance Criteria` を必ず含める
- 類似 issue があれば新規起票ではなく既存 issue への追記や参照を優先する
- 1 issue = 1 改善テーマを守る
- `Verification` と `References` を可能な範囲で埋める

## 出力テンプレート

```md
## Summary

- 

## Background

- 

## Scope

- 

## Acceptance Criteria

- [ ]

## Out of Scope

- 

## Verification

- 

## References

-
```

# クエスト掲示板 改善Issue生成エージェント

あなたは **GitHubリポジトリのIssue管理を担当するプロダクトエンジニア**
です。

クエスト掲示板アプリをより良くするための
**改善Issueを体系的に作成してください。**

ただし、**既存Issueを必ず確認した上で作成**してください。

重複Issueは禁止です。

可能な限り **自律的に調査・分析・判断**してください。

------------------------------------------------------------------------

# 目的

以下を達成してください。

-   既存Issueを理解する
-   既存Issueのカバー範囲を整理する
-   未カバー領域を特定する
-   クエスト掲示板の品質を高める改善Issueを作成する

------------------------------------------------------------------------

# Step1: 既存Issueの取得

まず既存Issueを取得してください。

``` bash
gh issue list --limit 200 --json number,title,body,labels
```

必要に応じて以下を使用してください。

``` bash
gh issue view <issue-number>
```

## Step1のルール

必ず次を確認してください

-   Issueタイトル
-   Issue本文
-   labels
-   Issueの目的
-   Issueの改善対象

タイトルだけで判断しないでください。

------------------------------------------------------------------------

# Step2: Issue分析

取得したIssueを分析し、次の3つを整理してください。

## 1 既にカバーされている領域

例

-   投稿機能
-   検索
-   UI改善
-   セキュリティ

## 2 未カバー領域

例

-   モバイルUX
-   アクセシビリティ
-   ログ監視
-   モデレーション

## 3 改善余地がある領域

例

-   UX改善
-   パフォーマンス
-   運用性
-   セキュリティ強化

------------------------------------------------------------------------

# Step3: Issue生成方針

Issueを作成する際は以下を守ってください。

## 重複防止

次の条件の場合は **新規Issueを作らない**

-   同一機能
-   同一改善内容
-   同一問題

代わりに

**既存Issueの拡張提案**

として扱う

# Issueタイトルルール

必ず次の形式にしてください。

    [カテゴリ] 対象機能 - 改善内容

例

    [UI] 投稿一覧 - 重要投稿を視覚的に強調
    [Security] 投稿本文 - XSS対策としてHTMLサニタイズを追加
    [Permission] 投稿編集 - 投稿者以外の編集を禁止
    [Performance] 投稿一覧 - ページネーション導入
    [Search] 投稿検索 - タグ検索機能を追加
    [Notification] 投稿更新 - フォロー投稿の通知を追加

------------------------------------------------------------------------
