
# AIドキュメント整理プロンプト

## 目的

このプロンプトは、リポジトリ内のAI関連ドキュメントを整理・再設計するためのものです。

対象ファイル:

- AGENTS.md
- prompt/agent.md
- docs/architecture.md
- docs/ai-execution.md
- README.md

目的は **AIエージェントが迷わず実装できるドキュメント構造に整理すること**です。

---

# あなたの役割

あなたは **AIエージェント設計とソフトウェアアーキテクチャに精通したシニアエンジニア**です。

このリポジトリの AI ドキュメントを以下の観点で再設計してください。

- AIが自走しやすい
- 重複ドキュメントがない
- 参照順序が明確
- 役割ごとに責務が分離されている
- Claude / Codex / Cursor などのエージェントでも使える

---

# 現在の状況

このリポジトリには以下のドキュメントが存在します。

- AGENTS.md
- prompt/agent.md
- docs/architecture.md
- docs/ai-execution.md
- README.md

しかし以下の問題がある可能性があります。

- 内容の重複
- 責務の曖昧さ
- AIが読む順序が不明確
- 実装ルールとアーキテクチャが分散

---

# タスク

以下を段階的に実行してください。

## Step1 現状分析

次の観点でドキュメントを分析してください。

- 役割
- 重複している内容
- AIが読むべき順序
- 不足しているドキュメント

分析結果をまとめてください。

---

## Step2 最適なドキュメント構造を設計

AIエージェントが使いやすい構造を設計してください。

例:

repo
├ README.md
├ AGENTS.md
├ docs
│ ├ architecture.md
│ ├ api.md
│ ├ database.md
│ └ ai-execution.md
└ prompt
   └ agent.md

このように

- 役割ごと
- AI参照順序

を明確にしてください。

---

## Step3 各ドキュメントの責務定義

以下のドキュメントの役割を明確にしてください。

README.md
AGENTS.md
architecture.md
ai-execution.md
prompt/agent.md

それぞれについて

- 目的
- AIが読むタイミング
- 含める内容
- 含めない内容

を整理してください。

---

## Step4 不要または統合すべき内容

重複している内容を整理してください。

例:

- 実装ルール
- アーキテクチャ説明
- AI実行フロー

どこに集約すべきかを提案してください。

---

## Step5 最終設計

最終的な構成を提示してください。

### ディレクトリ構成

### 各ファイルの責務

### AIの参照順序

例

1 README.md  
2 AGENTS.md  
3 docs/architecture.md  
4 docs/ai-execution.md  
5 ソースコード

---

## Step6 改善されたドキュメントテンプレート

以下のテンプレートを作成してください。

- AGENTS.md
- docs/architecture.md
- docs/ai-execution.md
- prompt/agent.md

AIがそのまま利用できる形式で作成してください。

# 出力形式

以下の構造で出力してください。

1 現状分析  
2 問題点  
3 改善案

5 各ドキュメントの責務  
6 AI参照順序

# 重要ルール

- 既存構造を完全に破壊しない
- AIエージェントが理解しやすい構造にする
- 冗長なドキュメントを減らす
- 実装ルールは一箇所に集約する
- 人間にも読みやすい設計にする

このプロンプトを実行した結果

- AIが迷わない
- ドキュメントの責務が明確
- 実装ルールが統一
- エージェントが自走できる

状態になること。
